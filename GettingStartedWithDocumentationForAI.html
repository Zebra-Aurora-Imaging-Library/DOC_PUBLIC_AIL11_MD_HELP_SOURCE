Getting Started: Aurora Imaging Library Documentation for AI

Last updated: May 11, 2026

To help you use AI tools (such as GitHub Copilot, Claude, ChatGPT, or Gemini) with Aurora Imaging Library, the documentation is published in two GitHub repos formatted for AI consumption:

Repo Format When to use it
DOC_PUBLIC_AIL11_MD Markdown (.md) files Recommended. Use whenever your AI tool can read a folder of files.
DOC_PUBLIC_AIL11_PDF PDF files Use only if your AI tool accepts PDF attachments exclusively.

Once you point an AI tool at one of these repos, you can ask natural-language questions about Aurora Imaging Library — for example, “How do I find a model and read a string?” — and get answers grounded in the documentation, including code samples in C, C++, C#, or Python. The sections below walk through how to set this up.

These repos are intended for AI consumption. For the official human-readable documentation, see the product support page at zebra.com/ail-info (Documentation tab).

Quick start — DOC_PUBLIC_AIL11_MD (recommended)

1. Clone the repo

git clone https://github.com/Zebra-Aurora-Imaging-Library/DOC_PUBLIC_AIL11_MD.git

2. Open the folder in an AI-enabled editor

Open the cloned DOC_PUBLIC_AIL11_MD folder in your editor's workspace so the AI can ground answers in it. In VS Code Copilot, you may also need to reference the folder explicitly with #codebase or attach it as chat context.

3. Choose a strong reasoning model

For the best results, use a strong reasoning model in your AI tool — for example, the latest Claude Opus, GPT, or Gemini Pro. Stronger models give more accurate code, better cross-module reasoning, and more useful citations back to the documentation.

4. Ask a question

Name the repo in your prompt so the AI looks there first:

Using DOC_PUBLIC_AIL11_MD, give me the steps to create an Aurora Imaging Library application that finds a model and reads a string. Include the Python code.

A graded set of prompts to try:

  • Trivial: “Using DOC_PUBLIC_AIL11_MD, what does MbufAlloc2d do?”
  • Typical: “Using DOC_PUBLIC_AIL11_MD, write a C# example that performs a 1D barcode read.”
  • Ambitious: “Using DOC_PUBLIC_AIL11_MD, summarize how to grab from a GenICam camera with MdigProcess, run a model find on each frame, and save matches to disk. Provide C++ code.”

The screenshot below shows GitHub Copilot configured to use DOC_PUBLIC_AIL11_MD with a strong reasoning model. Select DOC_PUBLIC_AIL11_MD — otherwise answers won't be grounded in the documentation.

Tips

  • Cite the repo by name in your first prompt of a session so the AI grounds its answers there.
  • Ask the AI to point to the file or section it used so you can verify.
  • Keep the cloned folder in your workspace for longer projects.

Alternatives

  • GitHub Copilot coding agent (github.com): No clone needed — pick DOC_PUBLIC_AIL11_MD from the repo dropdown.
  • Chat-tool workflow (no AI-enabled editor): See below.

Chat-tool workflow (ChatGPT, Claude.ai, Gemini, Copilot Chat web)

Most chat tools won't accept a whole repo, so attach only the files you need.

  1. Pick the files. Browse DOC_PUBLIC_AIL11_MD on GitHub. A good starting set is the User Guide overview file plus the Reference/<Module>/<Module>.md for whatever module you're using (Mbuf, Mim, MdigProcess, etc.).
  2. Attach the .md files to a new chat (drag-and-drop or Attach file).
  3. Ask your question, naming the files:

    Using the attached UserGuide.md and MdigProcess.md, give me a C++ example that grabs continuously from a GenICam camera and processes each frame.

If your tool accepts PDFs only, use DOC_PUBLIC_AIL11_PDF (clone or download from the GitHub UI) and attach the relevant PDFs instead. The repo is organized as one PDF per guide or module: UserGuide, Reference/<Module>, Examples, BoardSpecificNotes, ReleaseNotes.

The chat-tool workflow gives the AI less context than the editor workflow, so prefer the editor workflow for bigger or open-ended questions.

What works well, what doesn't

This is a public preview. Based on early use:

  • Works well: API lookups, code generation in C/C++/C#/Python, cross-module “how do I...” questions, comparing related functions.
  • Less reliable: exact numerical behavior of operations, very recent additions, questions that depend on board-specific notes you didn't attach.
