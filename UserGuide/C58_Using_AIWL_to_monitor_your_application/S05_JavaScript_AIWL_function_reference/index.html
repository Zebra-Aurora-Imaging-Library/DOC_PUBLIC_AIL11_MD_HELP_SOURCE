JavaScript Aurora Imaging Web Library function reference

The following functions are available for creating an Aurora Imaging Web Library client with JavaScript.

To use these functions, you must load ail.js before loading your Aurora Imaging Web Library client application.

AIWL.Mapp​

The functions prefixed with AIWL.Mapp make up the Aurora Imaging Web Library Application module. The Application module allows you to connect to an Aurora Imaging Web Library server, inquire the client-side Aurora Imaging Library identifiers of objects published by an server, and hook functions to connection and publishing events.

AIWL.MappCloseConnection​

This function closes a connection to an Aurora Imaging Web Library server application opened by AIWL.MappOpenConnection.

Note: AIWL.MappCloseConnectionis executed asynchronously; this function returns, and execution of your program continues, before the connection is closed. You can hook to the disconnection event using AIWL.MappHookFunctionwithAIWL.M_DISCONNECT.

This function is similar, but not identical, to the Aurora Imaging Library functionMappCloseConnection.

[Image: z_TableSpacer20x20.png]voidAIWL.MappCloseConnection(numberApplicationId)
ApplicationIdSpecifies the client-side Aurora Imaging Library application context identifier of the Aurora Imaging Web Library server application from which to disconnect. The application identifier must have been previously obtained, typically when connecting to the server application usingAIWL.MappOpenConnection.

AIWL.MappControl​

This function controls a specified application setting of this instance of the Aurora Imaging Web Library client application, or closes a connection to an Aurora Imaging Library object.

Note: For some settings,AIWL.MappControlis executed asynchronously; this function returns, and execution of your program continues, before the operation is complete. Refer to an individual setting to learn if it is asynchronous and, if so, how to hook to the operation completion event.

This function is similar, but not identical, to the Aurora Imaging Library functionMappControl.

[Image: z_TableSpacer20x20.png]voidAIWL.MappControl(numberApplicationId,numberControlType,numberControlValue)
ApplicationIdThis parameter is reserved for future expansion and must be set toAIWL.M_DEFAULT.
ControlTypeSpecifies the type of application setting to control. This parameter can be set to one of the following values:
AIWL.M_CLOSE_CONNECTIONCloses the connection to the Aurora Imaging Library object, specified by the ControlValue parameter. This operation is completed asynchronously. You can hook to the disconnection event using AIWL.MobjHookFunction with AIWL.M_DISCONNECT. Once the connection is closed (asynchronously), you will not be able to access the specified object until you reinquire its client-side Aurora Imaging Library identifier using AIWL.MappInquireConnection with AIWL.M_OBJECT_ID and a new connection to the object is established (asynchronously).
AIWL.M_ERRORControls whether to print error messages generated by this instance of the Aurora Imaging Web Library client application to the developer console of the web browser. Note that this does not print standard Aurora Imaging Library error messages (generated by the Aurora Imaging Web Library server application). In this case, set ControlValue to one of the following values:
AIWL.M_PRINT_DISABLESpecifies not to print Aurora Imaging Web Library error messages to the developer console of the web browser. If error printing is disabled, you can still check for Aurora Imaging Web Library errors. To do so, hook a function to application errors using AIWL.MappHookFunctionwith AIWL.M_ERROR. You can retrieve the error code within the hook-handler function, using AIWL.MappGetHookInfo with AIWL.M_ERROR.
AIWL.M_PRINT_ENABLESpecifies to print Aurora Imaging Web Library error messages to the developer console of the web browser. This is the default value.
ControlValueSpecifies the setting's new value, or the identifier of the Aurora Imaging Library object for which to close the connection.

AIWL.MappGetHookInfo​

This function retrieves information about the event that caused the hook-handler function to be called. This function should only be called within the scope of an application hook-handler function call (see AIWL.MappHookFunction).

The returned value is the requested information.

This function is similar, but not identical, to the Aurora Imaging Library functionMappGetHookInfo.

[Image: z_TableSpacer20x20.png]numberAIWL.MappGetHookInfo(numberApplicationId,numberEventId,numberInfoType,object**UserVar **)
ApplicationIdThis parameter is reserved for future expansion and must be set toAIWL.M_DEFAULT.
EventIdSpecifies the client-side Aurora Imaging Library application event identifier that was received by the hook-handler function.
InfoTypeSpecifies the type of information to get. This parameter can be set to the following value:
AIWL.M_OBJECT_IDRetrieves the client-side Aurora Imaging Library identifier of the Aurora Imaging Web Library server application or Aurora Imaging Library object that generated the event. > Note: Note that this client-side Aurora Imaging Library identifier should only be used within the instance of your Aurora Imaging Web Library client application that called this function. The server application, and other instances of your client application, use a different Aurora Imaging Library identifier to refer to the same object. In this case, this function returns the client-side Aurora Imaging Library identifier as a number.
**UserVar **Specifies the JavaScript object in which to write the requested information. The information is written to thedata property of the object. Since this function also returns the requested information, you can set this parameter to AIWL.M_NULL.

AIWL.MappHookFunction​

This function allows you to attach or detach a user-defined function to a specified application event. Once a hook-handler function is defined and hooked to an event, it is automatically called when the event occurs.

This function is similar, but not identical, to the Aurora Imaging Library functionMappHookFunction.

[Image: z_TableSpacer20x20.png]voidAIWL.MappHookFunction(numberApplicationId,numberHookType,stringHookHandler,object**UserData **)
ApplicationIdSpecifies the client-side Aurora Imaging Library identifier of the Aurora Imaging Web Library server application to which to hook the function. The application identifier must have been previously obtained, typically when connecting to the server application usingAIWL.MappOpenConnection.
HookTypeSpecifies the event type. This parameter can be set to one of the following values. > Note: To detach (unhook) a function that you previously hooked, combine the event type withAIWL.M_UNHOOK (for example, AIWL.M_CONNECT + AIWL.M_UNHOOK).
AIWL.M_CONNECTSpecifies to call the hook-handler function when your Aurora Imaging Web Library client application establishes a connection to the specified Aurora Imaging Web Library server application (or one of its published Aurora Imaging Library objects). You can useAIWL.MappGetHookInfowith AIWL.M_OBJECT_IDto learn which application or object connected, and AIWL.MobjInquirewith AIWL.M_OBJECT_TYPEto learn the type of the application or object.
AIWL.M_DISCONNECTSpecifies to call the hook-handler function when your Aurora Imaging Web Library client application disconnects from the specified Aurora Imaging Web Library server application (or one of its published Aurora Imaging Library objects). You can useAIWL.MappGetHookInfowith AIWL.M_OBJECT_IDto learn which application or object disconnected, and AIWL.MobjInquirewith AIWL.M_OBJECT_TYPEto learn the type of the application or object.
AIWL.M_ERRORSpecifies to call the hook-handler function when an error occurs while attempting to connect to the Aurora Imaging Web Library server application or published Aurora Imaging Library object.
AIWL.M_OBJECT_PUBLISH_WEB Specifies to call the hook-handler function when the Aurora Imaging Web Library server application publishes or unpublishes an Aurora Imaging Library object.
HookHandlerSpecifies the name of the JavaScript function that should be called when the event occurs. This function should have the following prototype: function HookHandler(HookType, EventId, UserVar) . For more information about hook-handler functions in JavaScript, see Aurora Imaging Library primer for JavaScript developers.
UserDataSpecifies the JavaScript object containing the user data that you want to make available to the hook-handler function. This object is passed to the hook-handler function, through its UserVar parameter, when the specified event occurs. Omit this parameter if not used.

AIWL.MappInquire​

This function inquires about a specified application setting of this instance of the Aurora Imaging Web Library client application, or a connected Aurora Imaging Web Library server application.

The returned value is the requested information.

This function is similar, but not identical, to the Aurora Imaging Library functionMappInquire.

[Image: z_TableSpacer20x20.png]numberAIWL.MappInquire(numberApplicationId,numberInquireType,object**UserVar **)
ApplicationIdSpecifies the client-side Aurora Imaging Library identifier of the Aurora Imaging Web Library server application about which to inquire the application settings. The application identifier must have been previously obtained, typically when connecting to the server application usingAIWL.MappOpenConnection. If not used, set this parameter toAIWL.M_DEFAULT.
InquireTypeSpecifies the type of application setting about which to inquire. This parameter can be set to one of the following values:
AIWL.M_ERRORInquires whether error messages generated by this instance of the Aurora Imaging Web Library client application are printed to the console of the web browser. In this case, the function returns one of the following values:
AIWL.M_PRINT_DISABLESpecifies that error messages will not be printed to the developer console of the web browser.
AIWL.M_PRINT_ENABLESpecifies that error messages will be printed to the developer console of the web browser.
AIWL.M_WEB_CLIENT_INDEXInquires the index used by the Aurora Imaging Web Library server application to identify this instance of the Aurora Imaging Web Library client application. > Note: This index is guaranteed not to change until this instance of the client application is disconnected. Note that this is not an iterable index; connected instances of your client are not guaranteed to be assigned sequential indices, and the indices do not shift when a client disconnects. In this case, this function returns the index as a number.
**UserVar **Specifies the JavaScript object in which to write the requested information. The information is written to thedata property of the object. Since this function also returns the requested information, you can set this parameter to AIWL.M_NULL.

AIWL.MappInquireConnection​

This function inquires about the Aurora Imaging Library web objects published by an Aurora Imaging Web Library server application.

The returned value is the requested information.

Note: For some settings,AIWL.MappInquireConnectionis executed asynchronously; this function returns, and execution of your program continues, before the operation is complete. Refer to an individual setting to learn if it is asynchronous and, if so, how to hook to the operation completion event.

This function is similar, but not identical, to the Aurora Imaging Library functionMappInquireConnection.

[Image: z_TableSpacer20x20.png]_number_or number[]AIWL.MappInquireConnection(numberApplicationId,numberInquireType,stringControlFlag,numberExtraFlag,objectUserVar)
ApplicationIdSpecifies the client-side Aurora Imaging Library identifier of the Aurora Imaging Web Library server application, about which to inquire information related to published objects. The application identifier must have been previously obtained, typically when connecting to the server application usingAIWL.MappOpenConnection.
InquireTypeSpecifies the type of published object information about which to inquire. This parameter can be set to one of the following values:
AIWL.M_WEB_PUBLISHED_LISTInquires the list of client-side Aurora Imaging Library identifiers associated with all Aurora Imaging Library objects that have been published by the Aurora Imaging Web Library server application. In this case, this function returns an array of client-side Aurora Imaging Library identifiers of the published objects as a number[]. > Note: Note that these client-side Aurora Imaging Library identifier should only be used within the instance of your Aurora Imaging Web Library client application that called this function. The server application, and other client applications, use different Aurora Imaging Library identifiers to refer to the same objects.
AIWL.M_WEB_PUBLISHED_LIST_SIZEInquires the current number of Aurora Imaging Library objects published by the Aurora Imaging Web Library server application. In this case, the function returns the number of published objects as a number.
AIWL.M_WEB_PUBLISHED_NAMEInquires the client-side Aurora Imaging Library identifier of a published Aurora Imaging Library object associated with the name specified by theControlFlagparameter. When inquiring the client-side Aurora Imaging Library identifier, this function also asynchronously creates a connection to the published object (if this connection had not already been made by a previous call to AIWL.MappInquireConnection). Therefore, the application identifier returned by this function is not valid at the time it is returned; you must not use it with any Aurora Imaging Web Library function except AIWL.MobjHookFunction. To know when the connection is made (or canceled due to an error), you can hook to these events using AIWL.MobjHookFunctionwithAIWL.M_CONNECT andAIWL.M_ERROR, respectively. After the AIWL.M_CONNECT event is generated, the client-side Aurora Imaging Library identifier is valid and can be used with other Aurora Imaging Web Library functions. In this case, this function returns the client-side Aurora Imaging Library identifier as a number. > Note: Note that this client-side Aurora Imaging Library identifier should only be used within the instance of your client application that called this function. The server application, and other instances of your client application, use a different Aurora Imaging Library identifier to refer to the same object.
ControlFlagSpecifies an attribute of the inquire operation to perform. If this parameter is not required for the inquire operation, it must be set to AIWL.M_DEFAULT.
ExtraFlagThe Extraflag parameter is reserved for future use and must be set to AIWL.M_NULL.
**UserVar **Specifies the JavaScript object in which to write the requested information. The information is written to thedata property of the object. Since this function also returns the requested information, you can set this parameter to AIWL.M_NULL.

AIWL.MappOpenConnection​

This function initiates a connection to a local or remote computer running an Aurora Imaging Web Library server application. It returns the client-side Aurora Imaging Library application context identifier of the server application, and optionally writes it in the data property of the object that you passed to UserVar.

Note: Note that this client-side Aurora Imaging Library identifier should only be used within the instance of the Aurora Imaging Web Library client application that called this function. The Aurora Imaging Web Library server application, and other Aurora Imaging Web Library client applications, use a different Aurora Imaging Library identifier to refer to the same application context.

The connection is created asynchronously, meaning that your program will continue to execute before the connection is established. Therefore, the application identifier returned by this function is not valid at the time it is returned; you must not use it with any Aurora Imaging Web Library function except AIWL.MappHookFunction. To know when the connection is made or canceled due to an error, you can hook to these events using AIWL.MappHookFunctionwithAIWL.M_CONNECT andAIWL.M_ERROR, respectively. After the AIWL.M_CONNECT event is generated, the Aurora Imaging Library identifier is valid and can be used with other Aurora Imaging Web Library functions.

Note: Note that AIWL.MappHookFunctionrequires that you pass the application identifier written to the **UserVar ** parameter by this function. Therefore, you cannot hook to the connection event until after you have called AIWL.MappOpenConnection (unless you hook to all application connection events by setting the ApplicationIdparameter of AIWL.MappHookFunction to AIWL.M_DEFAULT). You should typically call AIWL.MappHookFunctionimmediately after AIWL.MappOpenConnectionto ensure that the connection or error event is caught.

You can close the connection to the Aurora Imaging Web Library server application using AIWL.MappCloseConnection in the Aurora Imaging Web Library client application.

This function is similar, but not identical, to the Aurora Imaging Library functionMappOpenConnection.

[Image: z_TableSpacer20x20.png]voidAIWL.MappOpenConnection(stringURL,numberInitFlag,numberControlFlag,object**UserVar **)
URLSpecifies the computer to which to connect, and if necessary, the listening port of the Aurora Imaging Web Library server application. You must specify the prefix "ws://", followed by the Hostname or IP address of the computer, optionally followed by the listening port number. For example, "ws://localhost:7861" or "ws://192.168.1.51:7861". > Note: You can always use the Hostname localhost to connect to an Aurora Imaging Web Library server application running on the same computer as the instance of the client application. In your Aurora Imaging Library application, you can set the listening port using MappControlwith M_WEB_CONNECTION_PORT. The default port is 7861. > Note: You might find it useful to create an Aurora Imaging Web Library client application that allows the user to enter the IP address or Hostname dynamically before connecting (for example, using a dialog box).
InitFlagThis parameter is reserved for future expansion and must be set to AIWL.M_DEFAULT.
ControlFlagThis parameter is reserved for future expansion and must be set to AIWL.M_DEFAULT.
**UserVar **Specifies the JavaScript object in which to write the Aurora Imaging Web Library server application identifier. The information is written to thedata property of the object.

AIWL.Mdisp​

The functions prefixed with AIWL.Mdisp make up the Aurora Imaging Web Library Display module. The Display module allows you to show a published 2D or 3D display in an HTML5 canvas, enable interactive manipulation of a published display, and hook functions to display events (such as user interaction).

Note: Unlike Aurora Imaging Library, Aurora Imaging Web Library uses the same module for both 2D and 3D displays. For a 3D display, only the final rendered image of the display is transmitted to your Aurora Imaging Web Library client application; 3D displays are therefore functionally equivalent to 2D displays from the perspective of an client. All functions and settings in this module are supported for both 2D and 3D displays, except where otherwise noted.

AIWL.MdispControl​

This function controls a specified display setting.

This function is similar, but not identical, to the Aurora Imaging Library functionMdispControl.

[Image: z_TableSpacer20x20.png]voidAIWL.MdispControl(numberDisplayId,numberControlType,numberControlValue)
DisplayIdSpecifies the client-side Aurora Imaging Library identifier of the display. This identifier must have been previously inquired, typically using AIWL.MappInquireConnectionwith M_WEB_PUBLISHED_NAME.
ControlTypeSpecifies the type of display setting to control. This parameter can be set to one of the values below:
AIWL.M_INTERACTIVESets whether the user can interact with the display. > Note: To learn the default keyboard controls, see M_KEYBOARD_USE. In this case, set ControlValue to one of the following values:
AIWL.M_DISABLESpecifies that the user cannot interact with the display. > Note: Note that this instance of your Aurora Imaging Web Library client application will still generate mouse and keyboard events for the purposes of hooked functions. However, the events are not sent to the Aurora Imaging Web Library server application or other instances of your client application. This is the default value.
AIWL.M_ENABLESpecifies that the user can interact with the display, using a keyboard and mouse. Only one connected instance at a time of the Aurora Imaging Web Library client application can enable this feature for the specified display. If another instance of the client application has this feature enabled for the specified display, this function has no effect.
AIWL.M_UPDATE_WEBSets whether the Aurora Imaging Web Library server application should send updated images of this display to this instance of the Aurora Imaging Web Library client application. > Note: To control updating of the display for all instances of the client application, use MdispControlwith M_UPDATE in the server application. In this case, set ControlValue to one of the following values:
AIWL.M_DISABLESpecifies not to send updated images of the display.
AIWL.M_ENABLESpecifies to send updated images of the display. Also, the display is forced to update immediately. This is the default value.
AIWL.M_NOWForces an immediate update of the display. In this case, the display is updated even if AIWL.M_UPDATE_WEB is set to AIWL.M_DISABLE.
ControlValueSpecifies the setting's new value.

AIWL.MdispInquire​

This function inquires about a specified display setting.

The returned value is the requested information.

This function is similar, but not identical, to the Aurora Imaging Library functionMdispInquire.

[Image: z_TableSpacer20x20.png]numberAIWL.MdispInquire(numberDisplayId,numberInquireType,object**UserVar **)
DisplayIdSpecifies the client-side Aurora Imaging Library identifier of the display. This identifier must have been previously inquired, typically using AIWL.MappInquireConnectionwith M_WEB_PUBLISHED_NAME.
InquireTypeSpecifies the type of display setting about which to inquire. This parameter can be set to one of the following values:
AIWL.M_IMAGE_HOST_ADDRESSInquires the pixel color values of the last bitmap image of the display transmitted by the Aurora Imaging Web Library server application. This information is returned as an array of type_number_, whereby the different color components of the pixel are stored in separate entries. > Note: For a 3D display, this bitmap image is the rendered output of the display. This is functionally equivalent to receiving the output of a 2D display; you can present both 2D and 3D displays using the same methods, including automatically usingAIWL.MdispSelectWindow. > Note: This setting does not return an address. It is namedAIWL.M_IMAGE_HOST_ADDRESSfor consistency with the AIWL C/C++ API. The pixel values are always returned in the RGBA packed format. The first 4 values store the color components of the first pixel (0,0) (starting with the R value), the next 4 values store the color components of the second pixel (1,0) and so on. You do not need to consider pitch when using this data. Typically, you should hook a function to the display update event using AIWL.MdispHookFunctionwith AIWL.M_UPDATE_WEB, so that you can inquire new image data when the contents of the display changes. In this case, this function returns the array of pixel values as a number[].
AIWL.M_INTERACTIVEInquires whether the user can interact with the display. > Note: This inquiry only returns information about this instance of your Aurora Imaging Web Library client application. If another instance of your client application is interactively controlling the specified display, this function returnsAIWL.M_DISABLE. In this case, the function returns one of the following number values:
M_DISABLESpecifies that the local user cannot interact with the display. This is the default value.
M_ENABLESpecifies that the local user can interact with the display, using a keyboard and mouse.

AIWL.M_SIZE_YInquires the height of the displayed image, in pixels. Note that this is unrelated to the size of the HTML5 canvas in which the display is presented. In this case, the function returns the height as a number.

AIWL.M_ENABLESpecifies to send updated images of the display.
**UserVar **Specifies the JavaScript object in which to write the requested information. The information is written to thedata property of the object. Since this function also returns the requested information, you can set this parameter to AIWL.M_NULL.

AIWL.MdispMessage​

This function generates a mouse or keyboard event for the specified display. This manipulates the display if interactive control is enabled (using AIWL.MdispControlwith AIWL.M_INTERACTIVE), and generates an event for the purposes of hooked functions. This allows you to simulate keyboard and mouse input.

Note: Note that if interactive control is not enabled, the event is only generated for this instance of your Aurora Imaging Web Library client application (not for the Aurora Imaging Web Library server application). The event is never generated for other instances of your client application.

This function is not similar to any Aurora Imaging Library function.

AIWL.M_KEY_UPSpecifies to generate an event as though a key was released. In this case, set EventValueto the JavaScript standard keycode of the key to simulate being released.

AIWL.M_MOUSE_LEFT_BUTTON_UPSpecifies to generate an event as though the left mouse button was released. In this case, set EventValuetoAIWL.M_NULL.
AIWL.M_MOUSE_MIDDLE_BUTTON_DOWNSpecifies to generate an event as though the middle mouse button was pressed. In this case, set EventValuetoAIWL.M_NULL.

AIWL.M_KEY_WINSpecifies the Windows key.

AIWL.MdispPan​

This function associates X- and Y-panning offsets with the specified display. The selected image buffer is displaced on the display, from the top-left corner of the HTML5 canvas.

Note: This pans the display for all instances of your Aurora Imaging Web Library client application. If you need to pan the image separately for each instance of your Aurora Imaging Web Library client application, you must allocate and publish a separate display for each instance and select the image to each display.

This function is not supported for 3D displays.

This function causes the Aurora Imaging Web Library server application to call the Aurora Imaging Library functionMdispPan.

[Image: z_TableSpacer20x20.png]voidAIWL.MdispPan(numberDisplayId,numberXOffset,numberYOffset)
DisplayIdSpecifies the client-side Aurora Imaging Library identifier of the 2D display. This identifier must have been previously inquired, typically using AIWL.MappInquireConnectionwith M_WEB_PUBLISHED_NAME.
XOffsetSpecifies the number of image pixels by which to pan an image buffer horizontally when it is displayed. Specify a positive offset value to displace the image to the left. This value can be a decimal fraction.
YOffsetSpecifies the number of image pixels by which to pan an image buffer vertically when it is displayed. Specify a positive offset value to displace the image upwards. This value can be a decimal fraction.

AIWL.Mobj​

The functions prefixed with AIWL.Mobj make up the Aurora Imaging Web Library Object module. The Object module allows you to read and write messages in Aurora Imaging Library message mailboxes, inquire the types of published objects, and hook functions to the update and connection events of published objects.

AIWL.MobjGetHookInfo​

This function retrieves information about the event that caused the hook-handler function to be called. This function should only be called within the scope of an object hook-handler function call (see AIWL.MobjHookFunction).

The returned value is the requested information.

This function is similar, but not identical, to the Aurora Imaging Library functionMobjGetHookInfo.

[Image: z_TableSpacer20x20.png]numberAIWL.MobjGetHookInfo(numberEventId,numberInfoType,objectUserVar)

InfoTypeSpecifies the type of information about which to inquire. This parameter can be set to the following value:
AIWL.M_OBJECT_IDRetrieves the client-side Aurora Imaging Library identifier of the object that generated the event. In this case, this function returns the client-side Aurora Imaging Library identifier as a number. > Note: Note that this client-side Aurora Imaging Library identifier should only be used within the instance of your Aurora Imaging Web Library client application that called this function. The Aurora Imaging Web Library server application, and other instances of your client application, use a different Aurora Imaging Library identifier to refer to the same object.
UserVarSpecifies the JavaScript object in which to write the requested information. The information is written to thedata property of the object. Since this function also returns the requested information, you can set this parameter to AIWL.M_NULL.
